# Emby Migrator v1.1.1

`v1.1.1` 完善了跨设备迁移时的导出包发现流程，并增加轻量的版本更新提示。

## 新增与改进

- 自动扫描 `/data/exports`、`/data/imports` 和可选挂载目录 `/imports` 中的有效导出包。
- 从旧设备复制完整导出包到宿主机 `/opt/emby-migrator/data/imports/` 后，页面可直接识别，无需手工输入容器路径。
- 导出任务完成后自动刷新导出包列表。
- 页面顶部版本号支持检查 GitHub 最新稳定版本，仅在发现新版本时显示紧凑提示。
- Docker 镜像增加 `/imports` 可选挂载点，并优化 Go 模块构建缓存层。
- README、部署示例、页面资源版本号统一更新至 `1.1.1`。

## 安全边界

- 只识别包含根目录 `manifest.json` 的完整导出包。
- 导入路径严格限制在已配置的导出包根目录内，并阻止符号链接和路径穿越逃逸。
- 版本检查失败不会影响登录、导出或导入功能；检查结果带缓存，避免频繁访问 GitHub API。

## 验证结果

- `go test -count=1 -buildvcs=false ./...` 通过。
- `go vet -buildvcs=false ./...` 通过。
- JavaScript 语法、Go 模块完整性、格式差异和公开文件敏感信息检查通过。
- Docker 三目录导出包扫描与版本检查冒烟测试通过。
