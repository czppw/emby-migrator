## 8. 已知边界

- `resume` 已验证“根据上次成功 report 跳过已完成项”的基础路径，但还不是完整状态级断点恢复，不能精确恢复到进行中的单张图片或头像上传。
- 增量导入当前依据导出包 manifest 的 skipped 标记跳过，不做目标库侧差异扫描。
- Docker 镜像/版本回滚不能撤销已经写入目标 Emby 的元数据、媒体图片或人物头像；目标 Emby 恢复依赖测试实例重建或 Emby 备份/快照。

## 9. 结论

版本 `0.1.0-beta.3` 的核心迁移路径、TG 通知、服务器地址簿、双版本兼容、增量导入包和基础 resume 续跑均已通过远端验证，并已发布到 GitHub / Docker Hub。建议新部署优先使用固定标签 `czppwa/emby-migrator:sha-97ba9d8` 或版本标签 `czppwa/emby-migrator:v0.1.0-beta.3`；回滚也使用固定 sha 标签，不依赖 `latest`。
